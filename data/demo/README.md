# Demo data

Small, fully-synthetic PLINK1 + master-file dataset for trying out GenomEn without
access to real genotype data. Generated by
[`docs/scripts/generate_demo_data.py`](../../docs/scripts/generate_demo_data.py)
(re-run it with different `--seed`/`--n_samples`/`--n_variants` to regenerate).

None of this is real biological data — genotypes are drawn independently per variant
from a random allele frequency, and the phenotype is a binary trait with a small
amount of injected genetic + covariate signal (~0.64 test ROC-AUC with the default
`docs/configs/demo.yml`), just enough to demo non-trivial predictive performance.

Contents (1000 samples x 5000 variants, ~1.6 MB total):

| File               | Description                                    |
|--------------------|-------------------------------------------------|
| `demo.bed/.bim/.fam` | PLINK1 binary genotype files (chromosomes 1-22) |
| `demo_master.phe`  | Tab-delimited phenotype/covariate/split file     |

`demo_master.phe` columns: `IID`, `population` (constant `"demo"`), `split`
(`train`/`val`/`test`), `age`, `sex`, `Global_PC1`-`Global_PC10`, and the binary
phenotype `PHENO1`.

## Usage

Point your `.env` at these files:

```bash
FAM_PATH="data/demo/demo.fam"
BED_PATH="data/demo/demo.bed"
BIM_PATH="data/demo/demo.bim"
MASTER_PATH="data/demo/demo_master.phe"
```

Then run the training script against the matching demo config:

```bash
python docs/scripts/train.py --cfg_path=docs/configs/demo.yml
```
